# HOOK ที่รับผิดชอบ

## 4. useMemo

### หลักการทำงาน

useMemo เป็น React Hook ที่ใช้เก็บค่าผลลัพธ์จากการคำนวณ เพื่อไม่ต้องคำนวณใหม่ทุกครั้งที่ Component Render จะคำนวณใหม่แค่ในตอนข้อมูลเปลี่ยนแปลง

### ตัวอย่างโค้ด

คำนวณราคารวมของสินค้าในตะกร้า

```tsx
const totalPrice = useMemo(() => {
  return cart.reduce(
    (sum, item) => sum + item.price * item.quantity,
    0
  );
}, [cart]);

กรองรายการสินค้าในหน้าเมนูตามชื่อสินค้าและหมวดหมู่

const filteredProducts = useMemo(() => {
  return products.filter((product) => {
    const matchSearch = product.name
      .toLowerCase()
      .includes(search.toLowerCase());

    const matchCategory =
      category === "All" || product.category === category;

    return matchSearch && matchCategory;
  });
}, [search, category]);
เหตุผลที่เลือกใช้

เลือกใช้ useMemo เพราะในระบบมีการคำนวณข้อมูล เช่น การหาราคารวมของสินค้า และการกรองเมนูสินค้า ถ้าข้อมูลไม่ได้เปลี่ยนก็ไม่ต้องคำนวณใหม่ทุกครั้ง ลดการทำงานระบบ

ข้อดี
ช่วยลดการคำนวณซ้ำ
ทำให้ระบบทำงานได้มีประสิทธิภาพมากขึ้น
เหมาะกับข้อมูลที่ต้องผ่านการคำนวณก่อนนำไปแสดงผล
ข้อจำกัด
ไม่จำเป็นต่อการคำนวณที่ง่ายใช้เวลาไม่นาน
ใช้มากไปโค้ดจะดูซับซ้อน จะงง
5. useCallback
หลักการทำงาน

useCallback เป็น React Hook ใช้จดจำฟังก์ชัน ไม่ให้สร้างฟังก์ชันใหม่ทุกครั้งที่ Component Render ฟังก์ชันจะถูกสร้างใหม่เมื่อค่าที่กำหนดไว้ใน Dependency เปลี่ยนแปลง

ตัวอย่างโค้ด
const addToCart = useCallback(
  (product: Product) => {
    dispatch({
      type: "ADD_TO_CART",
      payload: product,
    });
  },
  []
);

ฟังก์ชันนี้ถูกส่งไปยัง Component ลูก

<ProductCard
  product={product}
  onAddToCart={addToCart}
/>
เหตุผลที่เลือกใช้

เลือกใช้ useCallback กับฟังก์ชัน addToCart เพราะฟังก์ชันนี้ถูกส่งไปให้ Component ProductCard ใช้งาน และ ProductCard มีการใช้ memo เพื่อช่วยลดการ Render ที่ไม่จำเป็น ถ้าไม่มี useCallback ฟังก์ชันจะถูกสร้างใหม่ทุกครั้งที่หน้าเว็บ Render ทำให้ Component ลูกอาจ Render ใหม่โดยไม่จำเป็น

ข้อดี
ลดการ Render ที่ไม่จำเป็นของ Component ลูก
ทำงานร่วมกับ React.memo ได้ดี
ช่วยจัดการฟังก์ชันที่ต้องส่งผ่าน Props
ข้อจำกัด
ไม่จำเป็นสำหรับฟังก์ชันทั่วไปที่ไม่ได้ส่งไป Component อื่น
ถ้าใช้เยอะเกินไปอาจทำให้โค้ดอ่านยากขึ้น
คำถามอภิปรายท้ายใบงาน
1. เหตุใดจึงเลือกใช้ useReducer แทน useState ในการจัดการตะกร้าสินค้า

เลือกใช้ useReducer เพราะระบบตะกร้าสินค้ามีหลายการทำงาน เช่น เพิ่มสินค้า ลบสินค้า เพิ่มจำนวน และลดจำนวน ซึ่งแต่ละอย่างมีเงื่อนไขในการเปลี่ยนข้อมูลที่แตกต่างกัน

ถ้าใช้ useState อาจต้องเขียนฟังก์ชันจัดการหลายส่วนภายใน Component ทำให้โค้ดยาวและดูแลยากขึ้น แต่ useReducer ช่วยรวม Logic ของตะกร้าไว้ที่เดียว โดยใช้ Action เป็นตัวกำหนดว่าต้องการเปลี่ยนข้อมูลแบบไหน เช่น ADD_TO_CART หรือ REMOVE_FROM_CART

ทำให้โค้ดเป็นระเบียบและเพิ่มความสามารถอื่นในอนาคตได้ง่ายขึ้น

2. useMemo ช่วยเพิ่มประสิทธิภาพของระบบอย่างไร

useMemo ช่วยลดการคำนวณที่ไม่จำเป็น โดยจะเก็บผลลัพธ์เดิมไว้ และคำนวณใหม่เมื่อข้อมูลที่เกี่ยวข้องมีการเปลี่ยนแปลง

ในโปรเจกต์นี้ใช้ useMemo สำหรับคำนวณราคารวมของสินค้าในตะกร้า และใช้กรองรายการสินค้าในหน้าเมนู

ถ้าไม่มี useMemo ทุกครั้งที่ Component Render ระบบอาจต้องคำนวณข้อมูลใหม่ทั้งหมด ถึงแม้ว่าข้อมูลเดิมจะยังไม่เปลี่ยน ทำให้เกิดการทำงานที่ไม่จำเป็น

3. useCallback มีประโยชน์ในสถานการณ์ใด และหากไม่ใช้จะเกิดผลอย่างไร

useCallback เหมาะกับกรณีที่ต้องส่งฟังก์ชันไปให้ Component อื่นผ่าน Props โดยเฉพาะ Component ที่มีการใช้ React.memo

ในโปรเจกต์นี้ใช้กับฟังก์ชัน addToCart ที่ส่งไปให้ ProductCard ใช้เพิ่มสินค้าเข้าตะกร้า

ถ้าไม่ใช้ useCallback ทุกครั้งที่ Component หลัก Render ฟังก์ชันจะถูกสร้างใหม่ ทำให้ Props เปลี่ยน และอาจทำให้ Component ลูก Render ใหม่โดยไม่จำเป็น

4. เปรียบเทียบการใช้ useContext กับการส่งข้อมูลผ่าน Props (Props Drilling)

การส่งข้อมูลแบบ Props คือการส่งข้อมูลจาก Component แม่ไปยัง Component ลูกตามลำดับ หากมีหลายระดับจะต้องส่งผ่าน Component ที่ไม่ได้ใช้งานข้อมูลนั้น ทำให้โค้ดจัดการยากขึ้น

ส่วน useContext ช่วยให้ Component ต่าง ๆ สามารถเข้าถึงข้อมูลร่วมกันได้โดยตรง โดยไม่ต้องส่ง Props หลายระดับ

ในโปรเจกต์นี้ใช้ CartContext เพื่อแชร์ข้อมูลตะกร้าสินค้า ทำให้หน้าเมนูและหน้าสรุปราคาสามารถใช้ข้อมูล cart ร่วมกันได้ง่ายขึ้น และลดความซับซ้อนของการส่งข้อมูลระหว่าง Component
